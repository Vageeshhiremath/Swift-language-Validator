# Swift Subset Syntax Validator

This project implements a **Lexer** and **Parser** for a subset of the **Swift** programming language using the **Python Lex-Yacc (PLY)** library. It is designed to validate the syntax of key Swift constructs, demonstrating fundamental concepts of formal language theory and compiler construction.

## üöÄ Features

The validator is capable of performing lexical and syntactic analysis for the following Swift language features:

- **Variables and Constants:** `let` and `var` declarations, with optional type annotation and initialization.

- **Structures (****`struct`****):** Declaration of custom data types.

- **Generics:** Support for type parameter lists (`<T: Constraint, U>`) in structures.

- **Protocols:** Minimal support for protocol declarations.

- **Error Handling:** Detects and reports both lexical and syntactic errors.

## üìù Grammar Overview

The parser is built upon a Context-Free Grammar (CFG) that defines the valid structure of the Swift subset.

### Key Grammar Rules

The grammar covers the structure of declarations, including:

- **Variable/Constant Declaration:** `var_declaration : LET/VAR ID type_annotation assignment`

- **Structure Declaration:** `struct_declaration : STRUCT ID generic_params where_clause_opt LBRACE property_declaration_list RBRACE`

- **Generic Parameters:** `generic_params : LT generic_parameter_list RANGLE`

- **Type Constraint:** `constraint : ID COLON ID` (e.g., `T: Hashable`)
